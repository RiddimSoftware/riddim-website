# Adding a Product

Product pages are generated by 11ty from one shared template and one data file.

## Add the Data

Add one object to `src/_data/products.json`.

Required fields:

- `slug`: output filename under `/products/`, without `.html`.
- `name`: product name shown in the hero.
- `seo.title`, `seo.description`, `seo.canonical`, `seo.ogImage`: head, canonical, Open Graph, and Twitter metadata.
- `icon` and `iconAlt`: hero image asset and accessible text.
- `tagline`: hero supporting copy.
- `primaryCta`: the main button. Use `external: true` for off-site links and `appStoreIcon: true` for App Store buttons.
- `story`: one short paragraph.
- `screenshots`: an array of `{ "src", "alt", "caption" }` objects. `caption` is optional.
- `features`: an array of short bullet strings.
- `secondaryLinks`: optional secondary buttons such as privacy, source, contact, or web app links.

Keep asset paths boring: put static assets at the repository root and reference them as `product.png`, not `../product.png`.

## Build and Check

Run:

```bash
npm run build
```

Confirm the new page exists at `_site/products/<slug>.html`.

For local review, run:

```bash
npm run dev
```

Open `http://localhost:8080/products/<slug>.html` and check the desktop and mobile layouts.

The template lives at `src/_includes/product-page.njk`; adding a normal product should only require the data file and assets, so the page should take less than 10 minutes when copy and images are ready.
